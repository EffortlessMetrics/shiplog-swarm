# Handoffs

This directory records closeouts for repo-control and release-prep lanes.

Use handoffs for facts that should survive chat history:

- what landed;
- which PRs and promotions carried it;
- which proof commands and CI lanes passed;
- which claims changed;
- which boundaries remain;
- what the next agent should do.

Do not use handoffs as product specs or release approval. Specs define behavior,
plans sequence PRs, support tiers define what may be claimed, and release
approval remains in release docs.
