# shiplog 0.10.0 - Release Decision

**Latest shipped release:** `v0.9.0`
**Release target:** `v0.10.0` source configuration ergonomics and LLM endpoint security
**Status:** release-prep PR pending final merged-main preflight

## Decision

Prepare `0.10.0` for release execution after the release-prep PR merges and
the final preflight passes from current `main`. The release is intentionally
bounded to the merged sources ergonomics feature, the HTTPS endpoint fix, and
the package and documentation updates needed to publish that state honestly.

## Evidence basis

- PR #619 added `sources list`, `sources enable`, and `sources disable` with
  local config-preserving tests and CLI documentation.
- PR #625 fixed the medium endpoint disclosure finding recorded by security
  report PR #623. Its focused HTTPS tests, Clippy, no-panic check, and routed
  CI passed before merge.
- PRs #617, #618, #620, #621, #622, #623, and #624 were merged as maintenance
  or security-report cleanup before release preparation.

## Release boundary

This decision does not claim that the `v0.10.0` tag, GitHub release,
crates.io publication, release assets, or install smoke already exist. Those
claims require the commands and public-state checks in the readiness ledger
and execution handoff.
