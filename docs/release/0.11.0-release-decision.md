# shiplog 0.11.0 - Release Decision

**Release target:** `v0.11.0`
**Status:** shipped on 2026-07-12

## Decision

The merged usability tranche shipped as `0.11.0`. This is a product and
distribution release, not a redesign: the receipt engine, JSON contracts,
read/write safety labels, and fail-closed sharing boundary remain unchanged.

## Release boundary

The release includes objective-scoped readiness, GitHub credential reuse,
no-argument home, `next`, `update`, quick `add`, packet-first output, default
`open`, direct-share preflight, `status --check`, versionless installers, and
the owned Homebrew and Scoop channels.

The release does not include `shiplog start`, non-GitHub provider connection
flows, OS-backed redaction-key setup, scheduling, a TUI, or multi-repository
workspaces. Those remain explicitly bounded follow-ups.

## Safety and CI decision

Share commands remain explicit and fail-closed. Secrets remain ephemeral and
are represented in receipts only by safe credential metadata. Ordinary proof
runs on Linux; macOS is retained for release assets and package validation,
and Codecov is not a required release gate.
