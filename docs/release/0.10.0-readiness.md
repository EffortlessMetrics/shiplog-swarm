# shiplog 0.10.0 - Release Readiness

**Planned tag:** `v0.10.0`
**Latest shipped release:** `v0.9.0`
**Status:** release-prep in progress; tag and publication not started
**Theme:** source configuration ergonomics and security hardening

## Included state

- `shiplog sources list` and `--json` show the seven canonical source keys
  without provider calls.
- `shiplog sources enable --source <name>` and `disable` preserve comments,
  provider records, and tokens while changing only `enabled`.
- The opt-in OpenAI-compatible LLM backend rejects malformed and non-HTTPS
  endpoints before building a client or sending an authorization header.
- Workspace and public package metadata are aligned to `0.10.0`.

## Execution after green preflight

```text
rtk git tag -a v0.10.0 -m "shiplog v0.10.0"
rtk git push origin v0.10.0
```

Watch `release.yml` for package proof, binary builds, draft release creation,
asset upload, validation, and release-mode integration tests. After those
checks pass, publish the crate and promote the GitHub release:

```text
rtk cargo publish -p shiplog --locked
rtk gh release edit v0.10.0 --draft=false --latest
```

## Post-release proof

```text
rtk cargo install shiplog --version 0.10.0 --locked --force
```
