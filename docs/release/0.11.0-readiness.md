# shiplog 0.11.0 - Release Readiness

**Release target:** `v0.11.0`
**Theme:** low-friction review readiness
**Status:** shipped on 2026-07-12

## Included state

- Objective-scoped setup readiness keeps missing share configuration from
  blocking local intake.
- GitHub can reuse supported environment credentials or an authenticated `gh`
  session without persisting secrets.
- `shiplog`, `shiplog next`, `shiplog update`, `shiplog add`, and default
  `shiplog open` provide the normal read-only, refresh, capture, and packet
  workflow.
- Direct manager/public share commands explain and verify their profile before
  rendering, while remaining fail-closed.
- Versionless installers, Homebrew, and Scoop provide prebuilt installation
  paths.
- Routine CI remains Linux-first; release builds retain Linux, Windows, and
  macOS asset coverage.
